## Fit

Use this skill when the topic is still messy, ambiguous, or emotionally noisy.

Strong fit:

- "Help me think through this."
- "Let's kick this around."
- "Brainstorm this with me."

If the user already has a clear position and wants pressure testing, suggest `steelman`.

## Opening Gate

1. Ask for the topic or opening ramble if it is missing.
2. Echo the user's opener in plain language.
3. Keep the echo mirror-only: no early challenge, cleanup, or hidden reframing.
4. Ask whether the echo sounds right.
5. Only begin the brainstorm loop after the user confirms or corrects the frame.

## Brainstorm Loop

Every active turn uses this exact structure:

```markdown
## Current State
[where the topic stands now]

## Pressure Points
[weak framing, contradictions, missing facts, assumptions, tensions]

## Three Questions
1. [framing question]
2. [pressure question]
3. [movement question]
```

Use [[question-quality.md]] for the question bar.

## Natural Pivots

Honor pivots like:

- go deeper
- broaden the topic
- challenge that
- summarize where we are
- what are we missing
- stop or wrap

Keep the next response organized around the active loop unless the user explicitly asks for a different output.
