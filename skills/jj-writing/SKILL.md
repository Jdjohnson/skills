# JJ Writing

Use this skill to help a user make writing easier: clarify the point, shape the structure, draft in plain language, and revise without polishing away the user's intent.

This skill creates or develops writing. It does not replace `humanizer`, which is for removing AI-writing tells from existing text while preserving meaning.

## Workflow

| Step | Purpose | Node |
|------|---------|------|
| 1 | Choose the writing lane and keep the workflow light | [workflow](nodes/workflow.md) |
| 2 | Capture the goal, audience, source material, and constraints | [intake](nodes/intake.md) |
| 3 | Build the structure before drafting when structure is unclear | [structure](nodes/structure.md) |
| 4 | Draft from the approved direction or best available assumptions | [draft](nodes/draft.md) |
| 5 | Revise for clarity, accuracy, and voice fit | [revision](nodes/revision.md) |
| 6 | Save only through approved Dot routes | [save routes](nodes/save-routes.md) |

## Rules

- Start with the point the user is trying to make.
- Preserve the user's intent, edge, and useful specificity.
- Ask only the questions needed to draft well.
- Use `resources/preferences/writing.md` when it exists.
- Treat pasted source material as source, not voice to imitate unless the user says otherwise.
- Keep client-facing, public, or externally shared output draft-only until explicit approval.
- Ask before writing or updating files unless the user explicitly asks to save or capture.
- Follow `ROUTES.md` for every durable output.
