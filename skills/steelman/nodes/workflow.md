## Fit

Use this skill when the user has an idea, argument, strategy, or decision that needs to become stronger before action.

If the topic is too foggy to state as an idea, suggest `brainstorm` first.

## Interview Contract

- Ask 2-4 questions per round.
- Questions must be non-obvious and surface hidden assumptions.
- Do not ask questions the user has already answered.
- If the user says "skip" or "you decide," make a reasonable assumption and flag it.
- After each round, reflect back what was learned before choosing the next angle.

## Round Pattern

1. Ask a focused set of questions.
2. Reflect the useful signal from the answers.
3. Name tensions, contradictions, blind spots, or tradeoffs.
4. Choose the next angle using [[angle-selection.md]].

## Adaptation

Sequence by idea type and risk. Skip irrelevant angles and go deeper where the answer quality still feels weak.
