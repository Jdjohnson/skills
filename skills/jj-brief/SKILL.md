# JJ Brief

Use this skill to help a user understand something fast and well enough to act.

## Workflow

| Step | Purpose | Node |
|------|---------|------|
| 1 | Identify the input type and source requirements | [source handling](nodes/source-handling.md) |
| 2 | Write the brief in the required format | [brief format](nodes/brief-format.md) |
| 3 | Stay available for focused follow-up questions | [Q&A mode](nodes/qa-mode.md) |
| 4 | Save only through approved Dot routes | [save routes](nodes/save-routes.md) |

## Rules

- Lead with the bottom line.
- Separate known facts from judgment.
- Tag confidence when making judgments.
- State gaps plainly.
- Keep it concise unless the user asks for depth.
- Use source material first; research only when needed.
- Ask before writing or updating files unless the user explicitly asks to save or capture.
- Follow `ROUTES.md` for every durable output.
