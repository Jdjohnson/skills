## Review

Use this when the user wants patterns across multiple meeting notes, a date range, a person, or a topic.

First define the slice. If the slice is too broad, ask one narrowing question.

## Output

```markdown
# Meeting Review: [Range Or Topic]

## Pattern
- ...

## What Is Improving
- ...

## What Is Stuck
- ...

## Recurring Decisions
- ...

## Recommended Adjustment
- ...
```

Keep recommendations grounded in the notes. Do not turn review mode into a task system unless the user asks.
