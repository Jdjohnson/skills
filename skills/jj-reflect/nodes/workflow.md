# Reflection Workflow

The job is to help Jarad see what his journal is already saying. Stay grounded,
plain, and specific.

## Intake

1. Resolve the scope:
   - explicit date, week, month, or range
   - current day from `MEMORY.md`
   - theme or pattern from Jarad's request
2. If scope is still ambiguous, choose a useful default:
   - today for "reflect"
   - current week for "what patterns are showing up"
   - last 30 days for broad recurring themes
3. Gather sources using [sources](sources.md).
4. Read enough raw material to support the reflection. Do not summarize every
   private detail; extract signals.

## Daily Reflection Prompts

For current-day or morning reflection, honor the active planning preferences:

- journal entry
- one thing about Laura, Ruby, and Gus
- best thing that happened yesterday

Use these as reflection anchors, not as a rigid form. If Jarad already provided
the material, reflect on it. If he has not, ask only the next missing question.

## Analysis Lens

Look for:

- repeated phrases, worries, wants, and avoided decisions
- energy gains and energy leaks
- family/presence themes that should shape the day
- work pressure that is crowding out recovery or clarity
- commitments that need gentler follow-through
- a next honest question, not a forced answer

Avoid:

- diagnosis
- therapy-speak
- generic self-improvement advice
- turning reflection into task triage
- converting private journal material into durable updates without approval

## Response Shape

Start with the point. Then give a compact reflection.

Default shape:

1. **What I notice:** the main pattern or tension.
2. **Evidence:** short source-grounded signals, with file paths when useful.
3. **What it might mean:** a careful interpretation, labeled as interpretation.
4. **One question:** the next useful question for Jarad.
5. **Tiny next move:** optional, only if the journal points to one.

For longer range/theme reflections, add:

- **Pattern over time**
- **What changed**
- **What keeps returning**

Keep the answer human. The goal is not to manufacture insight; it is to make
the useful signal easier to see.
