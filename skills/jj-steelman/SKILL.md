# JJ Steelman

Use this skill to make an idea stronger by testing it fairly and directly.

## Workflow

| Step | Purpose | Node |
|------|---------|------|
| 1 | Run interview rounds and synthesize between rounds | [workflow](nodes/workflow.md) |
| 2 | Select useful pressure-test angles | [angles](nodes/angles.md) |
| 3 | Ask high-leverage questions | [question quality](nodes/question-quality.md) |
| 4 | Handle pivots and quick commands | [quick commands](nodes/quick-commands.md) |
| 5 | Wrap when enough has been tested | [wrap and synthesize](nodes/wrap-and-synthesize.md) |
| 6 | Produce an optional routed steelman brief | [brief output](nodes/brief-output.md) |

## Rules

- Start by restating the idea as fairly as possible.
- Test assumptions, incentives, risks, second-order effects, and alternatives.
- Ask two to four questions per round.
- After each round, synthesize what changed before asking more.
- Cover relevant angles, not every possible angle.
- Be direct without becoming performative.
- Ask before writing a final brief unless the user explicitly asked to save or capture it.
- Follow `ROUTES.md` for every durable output.
