# Brief Output

Create a steelman brief only when the user asks or approves.

Default structure:

- Idea
- Why It Matters
- Strongest Case For
- Strongest Case Against
- Key Assumptions
- Tradeoffs
- Open Questions
- Success Criteria
- First Steps

Route the brief to the nearest relevant project `dot/` folder or timeline day `dot/` folder. Do not write outside routed Dot folders.
